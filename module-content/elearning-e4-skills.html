Module 1 van 4

Wat is een Skill?

Een Skill is een herbruikbaar instructiepakket voor een specifieke taak. Je schrijft een keer op wat Claude moet doen, stap voor stap, met regels en voorkeuren, en slaat dat op in een SKILL.md-bestand. Claude Code kan Skills laden via slash-commando's (bijv. /skill-naam) of automatisch via de Skill tool wanneer de taakomschrijving overeenkomt met de skill-beschrijving. Je kunt Skills ook handmatig activeren door de naam van de Skill expliciet te noemen in je opdracht aan Claude Code, bijvoorbeeld: "Gebruik de review-skill om dit rapport te beoordelen."

Vergelijk het met een werkprocedure in een bedrijf: je schrijft een keer op hoe iets gedaan moet worden, en iedereen, inclusief Claude, volgt die procedure.

Module 2 van 4

Hoe ziet een SKILL.md eruit?

De structuur van een SKILL.md

1

Naam en beschrijving (bovenaan)

De naam en beschrijving tussen streepjes. Claude gebruikt de beschrijving om te bepalen of de Skill relevant is voor een opdracht.

2

Stappen

Genummerde lijst van wat Claude moet doen, in de juiste volgorde.

3

Regels

Wat altijd moet gelden: taal, toon, limieten, verboden acties.

Tip

De beschrijving bovenaan is cruciaal. Een goede beschrijving zorgt dat de Skill op het juiste moment wordt geladen. Wees specifiek over wanneer de Skill gebruikt moet worden.

Praktijksituatie

Je verwerkt elke week bezichtigingsrapporten op dezelfde manier: samenvatten in vijf punten, opslaan met een specifieke bestandsnaam, toevoegen aan een overzichtsbestand. Wat is de beste aanpak?

Skills opslaan en laden

Skills sla je op in de map .claude/skills/ in je projectmap. Claude Code kan de juiste Skill laden via slash-commando's, automatisch via de Skill tool wanneer de taakomschrijving matcht, of je activeert een Skill handmatig.
