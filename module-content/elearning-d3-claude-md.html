D3 - CLAUDE.md: projectgeheugen voor Claude Code

Elke sessie met Claude Code begint blanco. CLAUDE.md verandert dat: een bestand in je projectmap dat Claude Code automatisch inleest en zo altijd de juiste context heeft.

Na deze module kun je:

  • Uitleggen hoe CLAUDE.md werkt en waarom het de consistentie verhoogt
  • Bepalen welke informatie in CLAUDE.md hoort en wat je weglaat
  • Een effectief CLAUDE.md aanmaken en onderhouden
  • De hierarchie van meerdere CLAUDE.md-bestanden begrijpen en toepassen
Geschatte tijd: 18 minuten
Onderdeel 1 van 4

Wat is CLAUDE.md en hoe werkt het?

Claude Code heeft een context window: alles wat hij weet tijdens een sessie staat in dat venster. Zodra je de terminal sluit, is die context weg. De volgende keer begint Claude Code opnieuw blanco.

CLAUDE.md lost dit op. Het is een gewoon tekstbestand dat je aanmaakt in je projectmap. Elke keer dat je Claude Code opstart in die map, leest hij dit bestand automatisch in en voegt de inhoud toe aan zijn context. Instructies, regels, voorkeuren, projectcontext: Claude Code weet het direct, zonder dat jij het elke keer opnieuw hoeft te vertellen.

Vergelijk het met een interne briefing die je aan een nieuwe medewerker geeft: dit zijn onze regels, dit is hoe wij werken, dit zijn de dingen die je altijd moet doen of nooit moet doen. Het verschil is dat Claude Code die briefing letterlijk elke sessie opnieuw leest.

Waarom dit werkt

Omdat CLAUDE.md bij elke sessie ingelezen wordt, zijn je instructies altijd actief. Je hoeft niet te vertrouwen op geheugen of herhaling. Claude Code handelt consistent, ook bij een nieuw gesprek of een nieuwe terminal-sessie.

Onderdeel 1 van 4

Waar staat het bestand?

CLAUDE.md staat in de hoofdmap van je project, ook wel de root of working directory genoemd. Dat is de map waar je Claude Code opstart. De bestandsnaam is altijd exact CLAUDE.md in hoofdletters, zonder spaties.

Locaties die Claude Code inleest

1

Projectmap (primair)

Het CLAUDE.md in de map waar je werkt. Dit is het bestand dat je het meeste gebruikt.

2

Bovenliggende map (parent)

Als er ook een CLAUDE.md staat in de map daarboven, leest Claude Code dat ook in. Handig voor organisatiebrede regels die voor meerdere projecten gelden.

3

.claude/CLAUDE.md

Claude Code leest ook een bestand in de verborgen map .claude/ binnen je project. Hier zet je persoonlijke voorkeuren die je niet in Git wilt opnemen.

Let op

CLAUDE.md in de submap wordt niet automatisch ingelezen als je in de hoofdmap werkt. Claude Code leest alleen het bestand in de huidige werkmap en eventuele bovenliggende mappen, niet in onderliggende mappen.

Onderdeel 2 van 4

De vier inhoudscategorieen

Een goed CLAUDE.md bevat vier soorten informatie. Elk type heeft een andere functie. Samen geven ze Claude Code het complete beeld dat hij nodig heeft om consistent en correct te werken.

De vier categorieen

1

Wie en wat

Wie ben jij, wat doet dit project? Claude Code weet niet automatisch dat je bij een makelaarskantoor werkt of dat dit project voor een specifieke klant is. Geef die context expliciet.

2

Technische context

Welke mappen, bestandsformaten en naamgevingsconventies gebruik je? Waar moeten bestanden naartoe? Wat zijn de mappenstructuren die Claude Code moet kennen?

3

Gedragsregels

Schrijfstijl, taal, toon, bevestigingen bij ingrijpende acties. Alles wat bepaalt hoe Claude Code zich gedraagt, niet alleen wat hij doet.

4

Nooit doen

Expliciete verboden. Wat mag Claude Code onder geen beding doen? Denk aan: nooit bestanden verwijderen zonder bevestiging, nooit buiten de aangewezen mappen werken.

Wat je NIET in CLAUDE.md zet

Geen wachtwoorden of API-sleutels. Geen informatie die elk gesprek anders is. Geen lange lijsten met uitzonderingen voor eenmalige situaties. CLAUDE.md is voor structurele context, niet voor sessiespecifieke instructies.

Onderdeel 3 van 4

CLAUDE.md aanmaken

Twee manieren

A

Via Claude Code zelf

Open Claude Code in je projectmap en geef de opdracht: "Maak een CLAUDE.md aan met de volgende instructies: [jouw instructies]". Claude Code maakt het bestand aan en vult het direct in.

B

Handmatig

Maak een nieuw tekstbestand in je tekstverwerker of code-editor. Schrijf je instructies in gewone tekst of Markdown. Sla het op als CLAUDE.md (hoofdletters, precies zo) in de hoofdmap van je project.

Tip

Gebruik methode A als je al weet wat erin moet: vraag Claude Code om het bestand zelf te schrijven op basis van een beschrijving van je werkwijze. Hij produceert een goed gestructureerde CLAUDE.md die je daarna aanpast.

Onderdeel 3 van 4

Veelgemaakte fouten

Te lang en te breed
Een CLAUDE.md van 500+ regels met alle denkbare uitzonderingen. Resultaat: Claude Code raakt in de war door tegenstrijdige of irrelevante instructies. Houd het onder de 200 regels met alleen wat je daadwerkelijk gebruikt.
Klik voor uitleg
Wachtwoorden of API-sleutels
CLAUDE.md is een gewoon tekstbestand. Als je het in Git opslaat, is het zichtbaar voor iedereen met toegang tot de repository. Zet nooit gevoelige gegevens in CLAUDE.md. Gebruik hiervoor .env-bestanden of een secrets manager.
Klik voor uitleg
Sessiespecifieke instructies
Instructies die alleen voor een specifieke taak gelden, niet voor alle sessies. Voorbeeld: "Verwerk vandaag de facturen van klant Jansen." Dit hoort in de prompt van die sessie, niet in CLAUDE.md.
Klik voor uitleg
Nooit bijwerken
Een CLAUDE.md die maanden oud is en niet meer overeenkomt met je huidige werkwijze. Claude Code volgt verouderde instructies even gehoorzaam als actuele. Behandel CLAUDE.md als levend document en pas het aan als je werkwijze verandert.

Onderdeel 4 van 4

Hierarchie van CLAUDE.md-bestanden

Claude Code leest niet alleen het CLAUDE.md in je huidige werkmap. Hij doorzoekt ook de bovenliggende mappen en leest elk CLAUDE.md-bestand dat hij tegenkomt. Dichter bij de werkmap heeft voorrang bij conflicterende instructies.

Hoe de hierarchie werkt

1

Bovenliggende map: organisatiebrede regels

Organisatiebrede regels die voor alle projecten gelden. Schrijfstijl, taal, algemene gedragsregels. Zet dit in het CLAUDE.md van de map daarboven.

2

Projectmap: projectspecifieke instructies

Context en regels die alleen voor dit project gelden. Klantinformatie, specifieke mappenstructuren, uitzonderingen op de organisatieregel.

3

.claude/CLAUDE.md: persoonlijke voorkeuren

Persoonlijke instructies die je niet wilt committen. Je voegt .claude/ toe aan .gitignore zodat dit bestand prive blijft.

Onderdeel 4 van 4

CLAUDE.md als teamtool

CLAUDE.md is een gewoon tekstbestand. Dat betekent dat je het kunt opslaan in Git, net als alle andere projectbestanden. Als je dat doet, deelt het hele team automatisch dezelfde context en gedragsregels voor Claude Code.

Dit is een van de meest onderschatte voordelen: een junior medewerker die voor het eerst in een project werkt, heeft direct dezelfde Claude Code-instructies als een senior die al maanden aan het project werkt. Er is geen mondelinge overdracht nodig.

Wat commit je, wat niet?

Commit in GitHoud lokaal (.gitignore)
Projectcontext, klantinformatiePersoonlijke schrijfstijlvoorkeuren
BestandsnaamconventiesLokale maplocaties
Gedragsregels voor het teamPersoonlijke snelkoppelingen
Verboden acties (nooit doen)Wachtwoorden of tokens (nooit in CLAUDE.md)
Onderdeel 4 van 4

Onderhoud als levend document

De meest effectieve manier om CLAUDE.md bij te houden: update het direct als je merkt dat Claude Code iets doet wat je niet wilt. Niet later, niet in een aparte taak, maar op het moment dat je het opmerkt.

Geef Claude Code de opdracht: "Voeg toe aan CLAUDE.md: [jouw instructie]." Hij past het bestand aan en de instructie is direct actief in de volgende sessie.

Praktijksituatie

Claude Code heeft zojuist een rapport opgeslagen als 'rapport_final_v3_definitief.txt'. Je had gewild dat hij JJJJ-MM-DD-onderwerp.txt gebruikte. Dit is de derde keer dat dit misgaat. Wat is de meest structurele aanpak?
