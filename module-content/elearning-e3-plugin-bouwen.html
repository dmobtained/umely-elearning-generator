Umely E-learning

E3 - Je eerste plugin bouwen

Kant-en-klare Connectors koppelen Claude aan systemen van anderen. Een plugin koppelt Claude aan wat jij hebt: je eigen klantenlijst, je interne procedures, je eigen kennis. In deze module bouw je je eerste plugin, zonder code te schrijven.

Na deze module kun je:

  • Uitleggen wat het verschil is tussen een Connector en een plugin
  • Aan Claude Code beschrijven wat je plugin moet doen
  • De gegenereerde plugin starten via een startscript
  • De plugin koppelen aan Claude via het configuratiebestand
  • Verifiëren of Claude de plugin correct gebruikt
± 20 minuten
Onderdeel 1 van 3

Connector vs. plugin: wat is het verschil?

Je hebt in E2 geleerd hoe je een Connector inschakelt. Een Connector is een koppeling die Anthropic heeft gebouwd: jij schakelt hem in via de Claude-instellingen, en Claude kan daarna Gmail, je agenda of Notion gebruiken.

Een plugin werkt anders. Jij bouwt de koppeling zelf - niet Anthropic. Dat klinkt ingewikkelder dan het is, want Claude Code schrijft de code voor je. Het enige wat jij doet: beschrijven wat Claude moet kunnen opzoeken.

Kenmerk Connector Plugin
Wie bouwt het? Anthropic Jij (met hulp van Claude Code)
Welke data? Bestaande diensten (Gmail, Notion, Slack) Jouw eigen data die nergens anders staat
Hoe inschakelen? Via Settings - klik op Connect Plugin genereren, starten en koppelen
Wanneer gebruiken? Als de tool al bestaat als Connector Als je eigen data wilt ontsluiten
Tip: wanneer kies je welk?

Gebruik een Connector als de tool al bestaat als Connector (Gmail, agenda, Notion, Slack). Gebruik een plugin als je eigen data wilt ontsluiten die nergens anders staat: je eigen klantenlijst, je interne FAQ, je bedrijfsprocedures, je tarieven.

Onderdeel 2 van 3

Claude Code schrijft alles

Claude Code genereert een MCP-server - dat is de technische verbinding tussen Claude en jouw data. Je hoeft niet te begrijpen hoe die werkt. Je beschrijft alleen wat Claude moet kunnen opzoeken. Claude Code schrijft vervolgens alle benodigde bestanden.

1

Open Claude Code

Start Claude Code op je computer. Zorg dat je in een lege werkmap staat, of maak een nieuwe map aan (bijvoorbeeld mijn-plugin) en navigeer daarheen.

2

Vul de prompt in

Kopieer de template hieronder en vul in wat jouw plugin moet weten. Plak de ingevulde prompt in Claude Code.

3

Sla de bestanden op

Claude Code genereert drie bestanden. Sla ze op in de map die je hebt aangemaakt. Onthoud het volledige pad naar die map - je hebt het nodig in de volgende stap.

Prompttemplate voor Claude Code

Maak een plugin in Python die Claude toegang geeft tot de volgende informatie:

[hier beschrijf jij wat de plugin moet weten - in gewone taal]

Genereer drie bestanden:
1. De servercode als plugin.py
2. Een startscript start-plugin.bat voor Windows
3. De exacte tekst die ik moet toevoegen aan mijn Claude-configuratiebestand
   claude_desktop_config.json, inclusief het volledige pad naar plugin.py

Gebruik de officiële MCP Python SDK. Als Python niet beschikbaar is, gebruik dan Node.js.
Let op: onthoud het mappad

Geef de map een herkenbare naam zoals mijn-plugin en zet hem ergens vaste neer, bijvoorbeeld op je bureaublad of in je Documenten-map. Je hebt het volledige pad nodig in de volgende stap. Als je de map later verplaatst, werkt de plugin niet meer.

Als Claude Code aangeeft dat Python noch Node.js beschikbaar is op je computer, vraag dan je IT-beheerder om Python te installeren via python.org.

Onderdeel 3 van 3

De plugin koppelen aan Claude

Je vertelt Claude waar de plugin staat via een configuratiebestand. Claude leest dat bestand bij het opstarten. Dit is de meest kritieke stap: als het configuratiebestand een fout bevat, start Claude niet correct op.

1

Configuratiebestand openen

Het configuratiebestand staat op:

C:\Users\[jouwgebruikersnaam]\AppData\Roaming\Claude\claude_desktop_config.json

Snelste manier: druk op Windows + R, typ %AppData%\Claude en druk op Enter. Als de map Claude niet bestaat, maak hem dan aan.

Open het bestand in Kladblok, Notepad++ of VS Code. Gebruik geen Word - die kan het bestand beschadigen. Als het bestand niet bestaat, maak het aan en zet er {} in.

2

Configuratieregel toevoegen

Claude Code heeft de exacte tekst al voor je gegenereerd. Kopieer die tekst en vervang de bestaande inhoud van het bestand ermee.

Leeg configuratiebestand (voor):

{}

Na toevoegen van de plugin (na):

{
  "mcpServers": {
    "mijn-plugin": {
      "command": "python",
      "args": ["C:\\Users\\naam\\mijn-plugin\\plugin.py"]
    }
  }
}
3

Opslaan en Claude herstarten

Sla het bestand op. Sluit de Claude desktop-app volledig. Open Claude opnieuw. Claude leest het configuratiebestand alleen bij het opstarten.

JSON is gevoelig voor fouten

Als er een komma of haakje mist, start Claude niet correct op. Als dat gebeurt: open Claude Code en vraag "Mijn configuratiebestand geeft een fout. Dit is de inhoud: [plak de inhoud]. Wat klopt er niet?" Claude Code corrigeert het direct.

Onderdeel 3 van 3

Plugin starten en verifiëren

1

Plugin starten

Ga naar de map waar je plugin staat. Dubbelklik op start-plugin.bat. Als Windows een beveiligingsmelding toont, klik dan op "Meer info" en vervolgens "Toch uitvoeren". Er opent een terminalvenster - dat venster moet open blijven. De plugin draait zolang dat venster open is.

2

Testen

Stel Claude een vraag over informatie die alleen in jouw plugin staat. Bijvoorbeeld: "Wat zijn de actieve woningen in ons systeem?" of "Wat zijn onze tarieven voor belastingaangifte?"

Als Claude antwoordt met informatie uit je plugin: de plugin werkt.

Herstelstap: plugin werkt niet

Je hebt de plugin gestart en Claude opnieuw gestart, maar Claude zegt dat hij de informatie niet kent. Wat doe je als eerste?

Je hebt alle stappen doorlopen: van het verschil tussen Connector en plugin tot het starten en verifiëren van je eigen plugin. Ga door naar de kennischeck, daarna test je je kennis in de afsluitquiz.
